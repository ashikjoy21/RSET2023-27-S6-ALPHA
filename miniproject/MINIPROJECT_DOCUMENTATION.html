1. Project Overview

What This Project Does

This is a full-stack weather monitoring application that fetches real-time weather data from an Ambient Weather station, stores it in MongoDB, and displays it through an interactive React dashboard with various visualization charts.

2. Backend Server

server.js - Express Server Entry Point

This is the main entry point that starts the Express server, connects to MongoDB, and sets up automatic data synchronization.

Key Concepts:
  • express() creates the application
  • mongoose.connect() connects to MongoDB
  • setInterval() schedules hourly data sync
  • app.use(cors()) enables cross-origin requests
📄 server.js
const express = require('express');
const cors = require('cors');
const dotenv = require('dotenv');
const weatherRoutes = require('./routes/weather');

const mongoose = require('mongoose');
const dbSync = require('./utils/db-sync');

dotenv.config();

// Connect to MongoDB
mongoose.connect(process.env.MONGODB_URI)
    .then(() => {
        console.log('Connected to MongoDB');
        // Start the hourly sync directly after connection
        // 3600000 ms = 1 hour
        setInterval(() => {
            dbSync.syncAllDevices();
        }, 3600000);

        // Optional: Run one immediately on startup
        dbSync.syncAllDevices();
    })
    .catch(err => console.error('MongoDB connection error:', err));

const app = express();
const PORT = process.env.PORT || 3000;

app.use(cors());
app.use(express.json());
app.use(express.static('public'));

// Routes
app.use('/api/weather', weatherRoutes);

app.get('/', (req, res) => {
    res.send('Ambient Weather API Backend is running');
});

app.listen(PORT, () => {
    console.log(`Server is running on http://localhost:${PORT}`);
});

How It Works

  1. Environment Setup: Loads environment variables from .env file
  2. Database Connection: Connects to MongoDB using the connection string
  3. Auto-Sync: After connecting, starts an hourly sync job to fetch new weather data
  4. Middleware: Sets up CORS and JSON parsing
  5. Routes: Mounts weather API routes at /api/weather

routes/weather.js - API Endpoints

This file defines all the REST API endpoints for fetching weather data. It handles multiple data resolutions (5-min, hourly, daily) for efficient chart rendering.

API Endpoints:
  • GET /api/weather - Get all devices with latest data
  • GET /api/weather/history - Get historical data (multi-resolution)
  • GET /api/weather/wind - Get wind data for polar chart
  • GET /api/weather/stats - Get database statistics
  • GET /api/weather/forecast - Get ML-based forecast
📄 routes/weather.js (Part 1 - Main Endpoints)
const express = require('express');
const router = express.Router();
const { spawn } = require('child_process');
const path = require('path');
const ambientWeather = require('../utils/ambient-weather');
const Weather = require('../models/Weather');

// GET /api/weather
// Fetches latest data from API and saves to DB
router.get('/', async (req, res) => {
    try {
        const devices = await ambientWeather.getDevices();

        // Save to MongoDB asynchronously
        devices.forEach(async (device) => {
            if (device.lastData && device.lastData.date) {
                try {
                    await Weather.updateOne(
                        { macAddress: device.macAddress, date: device.lastData.date },
                        {
                            macAddress: device.macAddress,
                            date: device.lastData.date,
                            data: device.lastData,
                            fetchedAt: new Date()
                        },
                        { upsert: true }
                    );
                } catch (dbError) {
                    console.error('Error saving to DB:', dbError.message);
                }
            }
        });

        res.json(devices);
    } catch (error) {
        res.status(500).json({ error: 'Failed to fetch weather data' });
    }
});

Understanding the Code

  • Async/Await: Modern JavaScript pattern for handling promises
  • updateOne with upsert: Updates if exists, inserts if not - prevents duplicates
  • Error Handling: Try-catch blocks ensure errors don't crash the server
📄 routes/weather.js (Part 2 - History Endpoint)
// GET /api/weather/history
// Multi-resolution endpoint for efficient chart rendering
// Query params:
//   - days: number of days to fetch (default 7)
//   - resolution: '5min' | 'hourly' | 'daily' (default 'daily')
router.get('/history', async (req, res) => {
    try {
        const { days = 7, resolution = 'daily' } = req.query;
        const daysAgo = new Date();
        daysAgo.setDate(daysAgo.getDate() - parseInt(days));

        // Get records from MongoDB
        const records = await Weather.find({
            date: { $gte: daysAgo }
        })
            .sort({ date: 1 }) // Oldest first for charts
            .lean();

        // Transform record to standard format
        const transformRecord = (record) => {
            const data = record.data || {};
            return {
                date: record.date,
                temp: data.tempf 
                    ? Math.round((data.tempf - 32) * 5 / 9) 
                    : null,
                humidity: data.humidity ?? null,
                wind: data.windspeedmph ?? null,
                pressure: data.baromrelin 
                    ? Math.round(data.baromrelin * 33.8639 * 10) / 10 
                    : null,
                rainfall: data.dailyrainin 
                    ? parseFloat((data.dailyrainin * 25.4).toFixed(1)) 
                    : 0,
            };
        };

        // Resolution: 5min (raw data)
        if (resolution === '5min') {
            const result = records.map(transformRecord);
            return res.json(result);
        }

        // Resolution: hourly (aggregate by hour)
        if (resolution === 'hourly') {
            const hourlyData = {};
            records.forEach(record => {
                const date = new Date(record.date);
                const hourKey = `${date.getFullYear()}-${date.getMonth()}-${date.getDate()}-${date.getHours()}`;
                
                if (!hourlyData[hourKey]) {
                    hourlyData[hourKey] = {
                        records: [],
                        date: new Date(date.getFullYear(), date.getMonth(), 
                                      date.getDate(), date.getHours())
                    };
                }
                hourlyData[hourKey].records.push(transformRecord(record));
            });

            // Average each hour
            const result = Object.values(hourlyData).map(({ records: recs, date }) => {
                const avg = (arr, key) => {
                    const valid = arr.map(r => r[key]).filter(v => v !== null);
                    return valid.length 
                        ? Math.round(valid.reduce((a, b) => a + b, 0) / valid.length * 10) / 10 
                        : null;
                };
                return {
                    date,
                    temp: avg(recs, 'temp'),
                    humidity: avg(recs, 'humidity'),
                    wind: avg(recs, 'wind'),
                    pressure: avg(recs, 'pressure'),
                    rainfall: avg(recs, 'rainfall'),
                };
            });

            return res.json(result.sort((a, b) => new Date(a.date) - new Date(b.date)));
        }

        // Resolution: daily (one reading per day)
        // ... similar aggregation logic
    } catch (error) {
        res.status(500).json({ error: 'Failed to fetch history' });
    }
});

models/Weather.js - MongoDB Schema

Defines the data structure for storing weather records in MongoDB using Mongoose.

📄 models/Weather.js
const mongoose = require('mongoose');

const weatherSchema = new mongoose.Schema({
    macAddress: {
        type: String,
        required: true,
        index: true
    },
    date: {
        type: Date,
        required: true,
        index: true
    },
    data: {
        type: Object,
        required: true
    },
    fetchedAt: {
        type: Date,
        default: Date.now
    }
});

// Compound index to ensure uniqueness of records per device/time
weatherSchema.index({ macAddress: 1, date: 1 }, { unique: true });

module.exports = mongoose.model('Weather', weatherSchema);

Schema Breakdown

  • macAddress: Unique identifier for the weather station device
  • date: Timestamp of the weather reading
  • data: Flexible object containing all weather metrics (temp, humidity, wind, etc.)
  • fetchedAt: When this record was fetched from the API
  • Compound Index: Prevents duplicate records for the same device at the same time

Handles communication with the Ambient Weather external API.

const axios = require('axios');
const dotenv = require('dotenv');

dotenv.config();

const BASE_URL = 'https://rt.ambientweather.net/v1';

const getDevices = async () => {
    const apiKey = process.env.AMBIENT_WEATHER_API_KEY;
    const applicationKey = process.env.AMBIENT_WEATHER_APPLICATION_KEY;

    if (!apiKey || !applicationKey) {
        throw new Error('Missing Ambient Weather API Key or Application Key');
    }

    try {
        const response = await axios.get(`${BASE_URL}/devices`, {
            params: { apiKey, applicationKey }
        });
        return response.data;
    } catch (error) {
        console.error('Error fetching devices:', error.message);
        throw error;
    }
};

const getDeviceData = async (macAddress, limit = 288, endDate) => {
    const apiKey = process.env.AMBIENT_WEATHER_API_KEY;
    const applicationKey = process.env.AMBIENT_WEATHER_APPLICATION_KEY;

    try {
        const params = { apiKey, applicationKey, limit };
        if (endDate) params.endDate = endDate;

        const response = await axios.get(`${BASE_URL}/devices/${macAddress}`, { params });
        return response.data;
    } catch (error) {
        console.error(`Error fetching device data:`, error.message);
        throw error;
    }
};

module.exports = { getDevices, getDeviceData };

utils/db-sync.js - Automatic Data Synchronization

Handles scheduled synchronization of weather data from the API to MongoDB.

📄 utils/db-sync.js
const ambientWeather = require('./ambient-weather');
const Weather = require('../models/Weather');

const syncAllDevices = async () => {
    try {
        console.log('Starting scheduled weather sync (high-resolution)...');
        const devices = await ambientWeather.getDevices();

        let totalSaved = 0;
        for (const device of devices) {
            try {
                // Fetch last 24 readings (~2 hours of data at 5-min intervals)
                // This overlap ensures no data is missed even if sync is delayed
                const historicalData = await ambientWeather.getDeviceData(
                    device.macAddress, 
                    24
                );

                if (Array.isArray(historicalData) && historicalData.length > 0) {
                    const bulkOps = historicalData.map(record => ({
                        updateOne: {
                            filter: { 
                                macAddress: device.macAddress, 
                                date: record.date 
                            },
                            update: {
                                macAddress: device.macAddress,
                                date: record.date,
                                data: record,
                                fetchedAt: new Date()
                            },
                            upsert: true
                        }
                    }));

                    const result = await Weather.bulkWrite(bulkOps);
                    totalSaved += (result.upsertedCount + result.modifiedCount);
                }
            } catch (dbError) {
                console.error(`Error syncing device:`, dbError.message);
            }

            // Wait 1.1s between devices to respect API rate limits
            await new Promise(resolve => setTimeout(resolve, 1100));
        }
        console.log(`Weather sync complete. Records processed: ${totalSaved}`);
    } catch (error) {
        console.error('Scheduled sync failed:', error.message);
    }
};

module.exports = { syncAllDevices };

Why Bulk Operations?

Performance: bulkWrite() sends multiple database operations in a single request, which is much faster than individual updateOne() calls. This is essential when processing hundreds of weather readings.

import React, { useState, useEffect } from "react";

// Retry function for robust API calls
async function fetchWithRetry(url, retries = 3, delay = 2000) {
  for (let i = 0; i < retries; i++) {
    try {
      const res = await fetch(url);
      if (res.ok) return res;
      await new Promise(resolve => setTimeout(resolve, delay * (i + 1)));
    } catch (err) {
      if (i === retries - 1) throw err;
      await new Promise(resolve => setTimeout(resolve, delay * (i + 1)));
    }
  }
  throw new Error('Failed after retries');
}

export function MainWeatherCard() {

  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

    // Refresh data every 60 seconds
    const interval = setInterval(fetchWeatherData, 60000);
    return () => clearInterval(interval);
  }, []);

  async function fetchWeatherData() {
    try {
      setLoading(true);
      setError(null);

      const response = await fetchWithRetry('http://localhost:3000/api/weather');
      const data = await response.json();

      if (data && data.length > 0) {
        const device = data[0];
        const lastData = device.lastData || {};

        // Convert Fahrenheit to Celsius

          ? ((lastData.tempf - 32) * 5 / 9).toFixed(1)
          : null;

        setWeatherData({
          location: device.info?.name || 'Weather Station',
          temperature: tempC,
          humidity: lastData.humidity,

            ? Math.round(lastData.baromrelin * 33.8639) 
            : null,
          wind: lastData.windspeedmph,
          lastUpdated: lastData.date ? new Date(lastData.date) : null,
        });
      }
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  }

  // ... render JSX
}

DiurnalHeatmap.jsx - Temperature Heatmap

Visualizes temperature patterns across hours of the day and days of the week.

📄 components/DiurnalHeatmap.jsx (Key Logic)
import { useMemo } from 'react';

export function DiurnalHeatmap({ data = [], metric = 'temp', title = 'Diurnal Heatmap' }) {

    // Process data into heatmap format
    const heatmapData = useMemo(() => {
        const processed = {};

        data.forEach(record => {
            if (!record.date) return;

            let dateObj = new Date(record.date);
            let hour = dateObj.getHours();
            
            const day = dateObj.toLocaleDateString('en-US', { 
                month: 'short', day: 'numeric' 
            });
            const value = record[metric];
            const key = `${day}-${hour}`;

            if (hour !== undefined) {
                processed[key] = { day, hour, value };
            }
        });

        return Object.values(processed);
    }, [data, metric]);

    // Color function - maps temperature to color
    function getColor(value, hasData) {
        if (!hasData || value === null) {
            return 'rgba(255, 255, 255, 0.03)';
        }

        const range = maxVal - minVal || 1;
        const ratio = (value - minVal) / range;

        // Cool (blue) to Hot (red) gradient
        if (ratio < 0.2) return `rgba(99, 102, 241, ${0.7 + ratio})`;   // Indigo
        if (ratio < 0.4) return `rgba(34, 211, 238, ${0.7 + ratio*0.5})`; // Cyan
        if (ratio < 0.6) return `rgba(34, 197, 94, ${0.8 + ratio*0.3})`;  // Green
        if (ratio < 0.8) return `rgba(250, 204, 21, ${0.85 + ratio*0.15})`; // Yellow
        return `rgba(239, 68, 68, ${0.9 + ratio * 0.1})`; // Red
    }

    // Render grid of cells
    return (
        <div className="bg-slate-900 rounded-3xl p-8">
            {days.map(day => (
                <div key={day} className="flex">
                    {hours.map(hour => (
                        <div
                            key={hour}
                            style={{ backgroundColor: getColor(getValue(day, hour), hasValue(day, hour)) }}
                            title={`${day} ${hour}:00 — ${value}°C`}
                        />
                    ))}
                </div>
            ))}
        </div>
    );
}

useMemo Hook

Performance Optimization: useMemo caches the computed result and only recalculates when dependencies (data, metric) change. This prevents expensive recalculations on every render.

WindPolarChart.jsx - Wind Direction Polar Chart

Displays wind direction and speed data on a polar (circular) chart using HTML5 Canvas.

📄 components/WindPolarChart.jsx (Key Logic)
import { useState, useEffect, useRef, useCallback } from 'react';

export function WindPolarChart({ title = "Wind Direction & Speed" }) {
    const canvasRef = useRef(null);
    const [windData, setWindData] = useState([]);
    const [selectedDate, setSelectedDate] = useState('');

    // Viridis-style color scale for wind speed
    const getSpeedColor = (speed, maxSpeed) => {
        const t = Math.min(speed / Math.max(maxSpeed, 20), 1);

        // Purple -> Blue -> Teal -> Green -> Yellow
        const colors = [
            { r: 68, g: 1, b: 84 },      // Purple (0)
            { r: 59, g: 82, b: 139 },    // Blue (0.25)
            { r: 33, g: 145, b: 140 },   // Teal (0.5)
            { r: 94, g: 201, b: 98 },    // Green (0.75)
            { r: 253, g: 231, b: 37 }    // Yellow (1)
        ];

        // Interpolate between colors
        const idx = t * (colors.length - 1);
        const lower = Math.floor(idx);
        const upper = Math.min(lower + 1, colors.length - 1);
        const blend = idx - lower;

        const r = Math.round(colors[lower].r + blend * (colors[upper].r - colors[lower].r));
        const g = Math.round(colors[lower].g + blend * (colors[upper].g - colors[lower].g));
        const b = Math.round(colors[lower].b + blend * (colors[upper].b - colors[lower].b));

        return `rgb(${r}, ${g}, ${b})`;
    };

    // Draw the polar chart
    const drawChart = useCallback(() => {
        const canvas = canvasRef.current;
        if (!canvas) return;

        const ctx = canvas.getContext('2d');
        const centerX = canvas.width / 2 - 40;
        const centerY = canvas.height / 2;
        const maxRadius = Math.min(centerX, centerY) - 40;

        // Clear canvas
        ctx.fillStyle = '#ffffff';
        ctx.fillRect(0, 0, canvas.width, canvas.height);

        // Draw concentric circles (speed rings)
        for (let i = 1; i <= 4; i++) {
            const radius = (i / 4) * maxRadius;
            ctx.beginPath();
            ctx.arc(centerX, centerY, radius, 0, 2 * Math.PI);
            ctx.strokeStyle = '#e2e8f0';
            ctx.stroke();
        }

        // Plot wind data points
        windData.forEach(point => {
            const radian = (point.winddir - 90) * Math.PI / 180;
            const radius = (point.windspeed / maxSpeed) * maxRadius;
            const x = centerX + radius * Math.cos(radian);
            const y = centerY + radius * Math.sin(radian);

            ctx.beginPath();
            ctx.arc(x, y, 4, 0, 2 * Math.PI);
            ctx.fillStyle = getSpeedColor(point.windspeed, maxSpeed);
            ctx.fill();
        });
    }, [windData]);

    // ... rest of component
}

Canvas Drawing Concepts

  • useRef: Creates a reference to the canvas DOM element
  • getContext('2d'): Gets the 2D drawing context
  • Polar coordinates: Convert angle + distance to x,y using cos/sin
  • useCallback: Memoizes the draw function to prevent unnecessary recreations

export function TemperatureChart({ title = "Temperature Analysis" }) {

Data Aggregation Pattern

Grouping by Date: The code uses an object as a dictionary where each key is a date string. For each data point, it updates the min/max values for that day. This is a common pattern for aggregating time-series data.
